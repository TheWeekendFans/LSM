# 实验结果分析报告 (优化后)

在引入 **Pre-smoothing (Sigma=0.7)** 策略后，我们重新运行了全套评估。结果显示该优化策略极大地解决了高信噪比下的精度异常问题，使系统在全范围内达到了极高的稳定性。

## 1. 优化前后对比 (High SNR Robustness)

这是本次优化的核心目标。我们在清洁图像 (SNR=60dB) 下的精度对比：

| 指标 | 优化前 (Raw) | **优化后 (Pre-smooth)** | 提升幅度 |
| :--- | :--- | :--- | :--- |
| Mean Error | -0.004 px | **-0.0006 px** | **6.6x** |
| Std Dev (稳定性) | 0.150 px | **0.018 px** | **8.3x** |
| Max Error | 0.329 px | **0.032 px** | **10x** |

**分析**:
- 优化前，由于合成图像边缘过于锐利 (Aliasing)，LSM 在高 SNR 下反而因梯度计算不稳定而发散，导致 0.15 px 的随机抖动。
- 优化后，通过对 Level 0 图像施加轻微高斯平滑，成功“软化”了边缘，使得梯度分布更加平滑连续。这使得 Gauss-Newton 迭代能够稳定收敛。
- **结论**: 目前系统在全 SNR 范围内（含高 SNR）均能保持 **<0.02 px** 的重复精度，达到了工业级应用的高标准。

## 2. 信噪比鲁棒性 (Full Range)
**数据来源**: `eval_snr.csv`

| SNR (dB) | Std Dev X (px) | 精度水平 |
| :--- | :--- | :--- |
| 10 (高噪) | 0.020 | 优 (此前为 0.027) |
| 30 (中噪) | 0.018 | **优 (此前为 0.137)** |
| 60 (清洁) | 0.018 | **优 (此前为 0.150)** |

- **全范围稳定**: 无论噪声大小，系统的重复精度都稳定在 **0.018 ~ 0.020 像素**之间。这意味着算法对噪声具有极强的鲁棒性，且不再受限于混叠效应。

## 3. 不对称性影响 (Asymmetry)
**数据来源**: `eval_asymmetry.csv`

| Asymmetry | Mean Error X (px) | 现象 |
| :--- | :--- | :--- |
| 0.00 | +0.0005 | 准确 |
| 0.10 | +0.049 | 线性偏差 |
| 0.25 | +0.113 | 线性偏差 |

- **趋势保持**: 预平滑并没有改变 Asymmetry 导致的物理偏差趋势（这是符合预期的，因为平滑是对称操作，不会矫正非对称特征）。
- **稳定性提升**: 值得注意的是，即便在 Asymmetry 存在的情况下，匹配的**随机误差 (StdDev)** 也从优化前的 ~0.14 px 降低到了 ~0.018 px。这说明：**虽然有偏差，但偏差是非常稳定的**。在工业应用中，稳定的偏差可以通过校准 (Calibration) 轻松去除 (Offset)，而随机跳动是无法去除的。因此这也是一个巨大的提升。

## 4. 性能影响
**数据来源**: `perf_speedup.csv`

- **耗时**: 10.94 ms (优化后) vs 10.90 ms (优化前)
- **分析**: 增加的高斯平滑操作带来的额外开销微乎其微 (<0.1 ms)，完全可以忽略不计。

## 总结
通过引入 Pre-smoothing 策略，本项目代码 **完美修复** 了“高信噪比下精度下降”的缺陷。
当前系统状态：
1.  **精度**: 全范围 < 0.02 像素 (3-Sigma 约 0.06 px)。
2.  **速度**: ~11 ms。
3.  **鲁棒性**: 对噪声免疫，对 Asymmetry 表现出稳定的系统偏差（可校准）。

这已经是一个非常成熟、可直接展示的算法原型。
