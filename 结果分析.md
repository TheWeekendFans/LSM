# 实验结果分析报告

基于生成的三个 CSV 文件 (`eval_snr.csv`, `eval_asymmetry.csv`, `perf_speedup.csv`)，我们对 Pyramid LSM 引擎的性能进行了深入分析。

## 1. 性能加速 (Speedup)
**数据来源**: `perf_speedup.csv`

| 模式 | 平均耗时 (ms) |
| :--- | :--- |
| 无 SIMD | 18.67 ms |
| 有 SIMD (AVX2) | 10.90 ms |

- **加速比**: **1.71x**
- **分析**: 开启 AVX2 指令集优化后，整体匹配速度提升了约 70%。这主要得益于双线性插值和法方程组装（点积、累加）的向量化。考虑到算法中仍有部分串行逻辑（如金字塔构建的内存分配、边界检查等），1.7倍的端到端加速比是一个符合预期的良好结果。

## 2. 不对称性影响 (Asymmetry Robustness)
**数据来源**: `eval_asymmetry.csv` (Fixed SNR = 30dB)

| Asymmetry | Mean Error X (px) | 趋势 |
| :--- | :--- | :--- |
| 0.00 | -0.001 | 接近 0 |
| 0.05 | +0.035 | 偏移增加 |
| 0.10 | +0.064 | 偏移增加 |
| 0.15 | +0.076 | 偏移增加 |
| 0.25 | +0.118 | 显著偏移 |

- **分析**:
    - Mark 的不对称性 (Asymmetry) 会直接导致匹配结果的**系统性偏差 (Bias)**。
    - 数据显示 `Mean Error X` 随 Asymmetry 增加而线性增长。这是因为 LSM 本质上是在寻找"重心"或"最佳重合点"，当 Mark 边缘不对称时，数学上的最佳重合点会偏离几何中心。
    - **结论**: 算法能敏锐地感知到 Asymmetry，但在高 Asymmetry 下需要引入额外的补偿算法（例如基于边缘权重的 Robust LSM）来校正这一物理偏差。

## 3. 信噪比鲁棒性 (SNR Robustness)
**数据来源**: `eval_snr.csv`

| SNR (dB) | Std Dev X (px) | 现象分析 |
| :--- | :--- | :--- |
| 10 (高噪) | 0.027 | **异常高精度** |
| 30 (中噪) | 0.137 | 精度下降 |
| 60 (清洁) | 0.150 | **精度最差** |

- **异常现象分析**:
    - 我们观察到一个反直觉的现象：**图像越干净 (SNR越高)，匹配精度反而越差（标准差越大）。**
    - **原因推测**: 这通常是由于**合成数据生成的混叠 (Aliasing) 效应**与**LSM 的梯度计算特性**冲突导致的。
        1.  **锐利边缘**: 我们生成的仿真图像（Level 0）是非常锐利的几何图形。在极高 SNR 下，图像边缘是非常陡峭的阶跃信号。
        2.  **梯度局部性**: 对于极锐利的边缘，图像梯度只存在于 1-2 个像素宽的边缘处，平坦区域梯度为 0。
        3.  **收敛困难**: 当初始猜测（从上一层金字塔传来）稍有偏差，落入"平坦区域"时，LSM 的 Gauss-Newton 迭代因为梯度为 0 而无法计算正确的更新量，或者因为梯度不连续而发生震荡。
        4.  **噪声的"抖动"作用**: 在低 SNR (如 10dB) 下，噪声实际上起到了一种"随机抖动 (Dithering)"或"正则化"的作用，它平滑了目标函数，或者让算法能"跳出"锐利边缘导致的局部极小值/零梯度陷阱。
- **改进建议**:
    - 在 Level 0 (原始分辨率) 匹配前，也应对图像进行轻微的**高斯平滑 (Gaussian Blur)**，以扩展边缘梯度的作用范围，改善线性化近似的稳定性。这将显著提升高 SNR 下的收敛精度。

## 总结
1.  **效率**: SIMD 优化效果显著，满足高速量测需求。
2.  **精度**: 算法对 Asymmetry 敏感，符合物理预期。
3.  **稳定性**: 在“完美”图像上表现不如“噪声”图像，揭示了算法对锐利边缘的梯度计算存在改进空间（建议引入预平滑）。
